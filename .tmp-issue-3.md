## Контекст

Старые админ-страницы `/admin/staff`, `/admin/services`, `/admin/portfolio` **удалены** — на бэке этих сущностей больше нет. Текущая навигация в [admin/layout.tsx](https://github.com/ueeet/sakura/blob/master/frontend/src/app/admin/layout.tsx) содержит только: Дашборд, Брони, Отзывы, Настройки.

Нужно создать новые админ-страницы под сущности нового бэка.

## API контракт

Полный — в [API_DOCS.md](https://github.com/ueeet/sakura/blob/master/API_DOCS.md). Все CRUD-эндпоинты требуют `Authorization: Bearer <accessToken>` (`api.get/post/put/delete` уже добавляют его автоматически).

Типы — в [frontend/src/lib/types.ts](https://github.com/ueeet/sakura/blob/master/frontend/src/lib/types.ts) (`Branch`, `SaunaCategory`, `Sauna`, `PriceSlot`, `Promotion`, `Booking`).

## Что нужно создать

### 1. `/admin/branches` — управление филиалами

**API:** `/branches` (CRUD)

**Поля формы:**
| Поле | Тип | Required |
|------|-----|----------|
| slug | string | да (для URL) |
| name | string | да |
| address | string | да |
| phone | string | да |
| description | textarea | нет |
| latitude | number | да |
| longitude | number | да |
| coverImage | url | нет (через `api.upload`) |
| tourUrl | url | нет (3D-тур iframe) |
| sortOrder | number | нет |
| isActive | bool | да |

---

### 2. `/admin/categories` — категории саун

**API:** `/categories?branchId=X` (CRUD)

Только для филиалов с категориями (например, complex-9 имеет `family` и `regular`). complex-50 — без категорий.

UX:
- Select "Филиал" → загружает категории
- Список категорий с drag-n-drop сортировкой (`sortOrder`)
- Форма: slug, name, sortOrder

---

### 3. `/admin/saunas` — самая большая страница

**API:** `/saunas` (CRUD), `/upload` для фото

Список с фильтрами: филиал, категория, тип. Таблица: фото-превью, название, тип, филиал, категория, активна.

**Форма с табами:**

#### Таб "Основное"
| Поле | Тип |
|------|-----|
| slug | string (auto: `{branch}-{category}-{N}`) |
| name | string |
| type | select: `russian` / `finnish` / `hamam` |
| typeLabel | string ("Русская сауна" / "Финская сауна" / "Турецкий хамам") |
| size | select: `small` / `large` |
| sizeLabel | string ("Малый зал" / "Большой зал") |
| description | textarea |
| capacity | number |
| area | number (м²) |
| poolSize | string ("3×4 м" или null если нет бассейна) |
| hasBBQ | checkbox |
| branchId | select из филиалов |
| categoryId | select (nullable) |
| sortOrder | number |
| isActive | checkbox |

#### Таб "Удобства"
Два массива строк (`amenities` + `extras`). UI типа тэгов:

```tsx
<TagInput label="Основные удобства" value={amenities} onChange={setAmenities} />
```

`amenities` — что входит в стоимость (Wi-Fi, бассейн, ТВ).
`extras` — допы (массажное кресло, мангал).

#### Таб "Фото"
- `mainImage` — одно главное фото (через `api.upload`)
- `images` — массив URL с drag-n-drop сортировкой и удалением

#### Таб "Бронирование"
| Поле | Default |
|------|---------|
| cleaningMinutes | 60 (для хаммама — 90) |
| minHours | 2 |
| openHour | 0 |
| closeHour | 24 |

`cleaningMinutes` — буфер уборки **после** каждой брони.

#### Таб "Цены"
Grid 2×3: `weekday`/`weekend` × `day`/`evening`/`night`. Inline-edit `pricePerHour`. Сохранение через `POST/PUT /prices`.

---

### 4. `/admin/prices` — управление ценами

Можно либо встроить в форму сауны (см. таб "Цены" выше), либо отдельной страницей с фильтром по сауне.

**API:** `/prices?saunaId=X` (CRUD)

UI: grid 2×3 (`dayType` × `timeSlot`), inplace-edit `pricePerHour`.

---

### 5. `/admin/promotions` — управление акциями

**API:** `/promotions/all` (admin), `/promotions/:id` (CRUD)

Список с бейджами активна/неактивна/просрочена + drag-n-drop сортировка.

**Форма:**
| Поле | Тип |
|------|-----|
| slug | string |
| title | string |
| description | textarea |
| note | string (доп. строка "Кроме праздников") |
| icon | select из lucide-icons (`flame`, `gift`, `cake`, `star`, `percent`) |
| image | url (опц.) |
| promoCode | string (опц.) |
| discount | number 0-100 (опц.) |
| startDate | datetime (опц., null = бессрочно) |
| endDate | datetime (опц.) |
| isActive | bool |
| sortOrder | number |

## Полезное

В [frontend/src/lib/api.ts](https://github.com/ueeet/sakura/blob/master/frontend/src/lib/api.ts) есть всё нужное:

```ts
import { api } from "@/lib/api";

await api.get<Branch[]>("/branches");
await api.post<Branch>("/branches", { slug, name, ... });
await api.put<Branch>("/branches/1", { ... });
await api.delete("/branches/1");

const { url } = await api.upload(file);  // multipart/form-data
```

JWT-токен и refresh обрабатываются автоматически.

## Подсказки по UX

- **Toast notifications** — `sonner` или `react-hot-toast`
- **Подтверждение удаления** — модалка вместо `confirm()`
- **SSE realtime** — на дашборде уже подписан `onSSE(loadStats)`. Для броней тоже подключи (новая бронь появится без F5)
- **Загрузка фото** — drag-n-drop зона + preview
- **Tag input** для amenities/extras — `react-tag-input` или свой через `onKeyDown=Enter`
