## 💳 Добавлен payment flow — обновление BookingPicker

Бэк теперь автоматически создаёт **демо-платёж** при создании брони (через mock-провайдера). При показе хозяину это выглядит как реальная YooKassa, но без списания денег. Когда хозяин одобрит — провайдер свапается одной env-переменной.

### Что изменилось в `POST /api/bookings`

**Запрос** — добавилось одно поле:
```json
{
  "clientName": "Иван",
  "phone": "+79991234567",
  "startAt": "2026-04-15T11:00:00",
  "endAt": "2026-04-15T14:00:00",
  "guests": 6,
  "branchId": 1,
  "saunaId": 1,
  "totalPrice": 4200,
  "paymentType": "deposit"  // ← НОВОЕ: "deposit" (30%) или "full" (100%)
}
```

**Ответ** — добавилось поле `payment` с `confirmationUrl`:
```json
{
  "id": 12,
  "clientName": "Иван",
  ...
  "status": "pending_payment",
  "paymentStatus": "pending",
  "totalPrice": 4200,
  "payment": {
    "id": 5,
    "externalId": "mock_xxxxx",
    "amount": 1260,
    "type": "deposit",
    "confirmationUrl": "http://localhost:3000/payment/mock_xxxxx?return_url=..."
  }
}
```

### Что должен сделать BookingPicker

**1. Перед отправкой формы** — добавить блок выбора типа оплаты:

```
┌─────────────────────────────────────┐
│ Сумма: 4200 ₽                       │
│                                     │
│ ⦿ Предоплата 30%        1260 ₽    │
│   Остальное на месте                │
│                                     │
│ ○ Полная оплата онлайн  4200 ₽    │
│                                     │
│ ☑ Согласен с офертой               │
│                                     │
│ [ Перейти к оплате → ]              │
└─────────────────────────────────────┘
```

`30%` берётся из `sauna.depositPercent` (по умолчанию 30, можно менять на сауну в админке).

**2. POST /bookings** с `paymentType` из выбора юзера

**3. После 201** — `window.location.href = response.payment.confirmationUrl`

```ts
const res = await fetch(`${API_URL}/bookings`, {
  method: "POST",
  body: JSON.stringify({ ..., paymentType: "deposit" }),
});
const data = await res.json();
window.location.href = data.payment.confirmationUrl;
// → редирект на /payment/[externalId] (демо-страница оплаты)
```

**4. Юзер на демо-странице оплаты** (`/payment/[externalId]`)
- Видит стилизованную под YooKassa форму с тестовыми данными карты
- Жмёт "Оплатить" → POST `/api/payments/mock/confirm` (имитация webhook)
- Редирект на `/booking/status?bookingId=X`

**5. Страница `/booking/status`** уже создана
- Polling `GET /api/bookings/:id/public` каждые 2 секунды
- Показывает финальный статус: ✅ "Оплачено" / ❌ "Отменено" / ⏳ "Ожидаем"

### Авто-отмена через 15 минут

Если юзер ушёл со страницы оплаты и не оплатил — cron на бэке через 15 минут:
- Меняет `Booking.status` → `"cancelled"`
- Освобождает слот
- Отменяет связанный Payment
- SSE broadcast `booking_deleted`

### Статусы Booking теперь

```
"pending_payment"  ← создана, ждёт оплаты (15 мин TTL)
"new"              ← оплачена, ждёт подтверждения админа
"confirmed"        ← подтверждена админом
"cancelled"        ← отменена
"completed"        ← завершена
```

### Поля для отображения статуса оплаты

```ts
booking.paymentStatus: "pending" | "deposit_paid" | "fully_paid" | "refunded"
booking.paidAmount: number  // в рублях, сколько уже оплачено
```

### Замена на реальную YooKassa (потом)

Когда хозяин одобрит — на стороне напарника **ничего менять не нужно**. Бэк-команда:
1. Регистрирует магазин в YooKassa
2. Создаёт `backend/src/lib/payment/yookassa.ts`
3. Меняет `PAYMENT_PROVIDER=yookassa` в `.env`
4. `confirmationUrl` начинает указывать на реальную страницу YooKassa

Тот же фронт-код продолжает работать.
